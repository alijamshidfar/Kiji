📋 What is the KAL Registry?

▼

The KAL Smart File Registry is a Google Sheets-based document control system that tracks every KAL Academy file across Google Drive.

It enforces a strict naming convention, validates each file against Drive, assigns visual audit status, and automates file creation, archiving, linking, and reporting — all from a single spreadsheet menu.

Sheets used: Registry (main data) · CODES & Descriptions (validation) · Settings (configuration)

🚀 Quick Start

▼
  • 1 Open the Registry Tab — each row represents a single file entry.
  • 2 Verify Codes — access the Show Codes menu to view valid DRIVE, ENTITY, and DOCTYPE tags.
  • 3 Format the Name — fill in Column C (File Name) using the pattern:
    [DRIVE CODE]-[ENTITY CODE]_[DOCTYPE CODE]_[Human-ReadableCamelCaseDescription]
  • 4 Audit & Sync — run KAL File Registry › Audit & Sync Selected File to validate your naming.
  • 5 Fix Diagnostics — resolve any red-highlighted errors flagged by the audit.
Note: New codes can be added to the "CODES & Descriptions" tab to register them in the system.

🎨 Row Status Colors

▼

After running an audit, each row is colored to show its status:

Red — Naming Error (Priority 1)
The file name fails structural or code validation. Column I (KAL Name Check) shows the exact error(s), e.g. Invalid Drive Code (XX).
Yellow — File Not Found (Priority 2)
Name is valid but no matching file was found in Google Drive. Create the file.
Green — Finalized (Priority 3)
Name is valid, file exists in Drive, and version is vFINAL.
No color — In Progress
Name is valid and file exists, but not yet finalized.

⚙️ Settings Tab Requirements

▼

The Settings sheet must have these cells configured before using advanced features:

A2 Default Destination Drive folder URL (used by Move to Destination Drive)
B2 Preferred KAL Template folder URL (used by Create Selected File)
C2 Archive folder URL (used by Archive Older Versions and Promote to Final)
Required: Paste the full Google Drive folder URL into each cell. Functions will error if the cell is empty.

📐 KAL Naming Convention

▼
[DRIVE]-[ENTITY]_[DOCTYPE]_[Description]_[YYYYMMDD]_v[N]
SegmentRulesExample
DRIVE 2–3 char code from CODES tab OP, LP, PC
ENTITY Org unit code from CODES tab KAL, STU
DOCTYPE Document type from CODES tab POL, RPT, FRM
Description CamelCase, no spaces OnboardingGuide
Date YYYYMMDD 20240315
Version v1, v2 … vFINAL v3, vFINAL

Full example:

OP-KAL_POL_DataPrivacyPolicy_20240315_vFINAL
Separator rules: Use - (hyphen) after the Drive code, _ (underscore) between all other segments. No spaces anywhere.

🤖 Abstract Column (L)

▼

Column L is auto-populated with an =AI() formula whenever a valid file name exists in col C:

=AI("Based ONLY on description '"&B2&"' and filename '"&C2&"', write a two-sentence summary.")

The formula uses the Gemini AI extension built into Google Sheets. If the cell shows an error, ensure the AI extension is installed and enabled for your account.

The Abstract is the rightmost data column intentionally — it can be hidden during data entry without affecting any registry data.

📁 File Operations

▼
Create Selected File Creates a new Google file in Drive for the selected row using the name in col C. Optionally copies from the template in col K or Settings!B2. Inserts the Drive link into col G.
Open Current Folder Opens the Drive folder listed in col F for the selected row in a new browser tab.
Move to Destination Drive Moves the selected file to the URL in col J (or Settings!A2 if col J is empty). Updates col F and col G automatically.
Promote to Final & Move Renames the file to vFINAL, moves it to Destination Drive, and archives the previous versioned copy to Settings!C2. Triggers a full row re-audit.
Remove Selected File Permanently deletes the Drive file linked in col G. Prompts for confirmation before deletion.
Remove All Versions Deletes all Drive files whose name matches the base name in col C (all version suffixes). Use with caution.

🔧 Maintenance

▼
Keep Only Latest Version Scans Drive for all versions of each registered file. Keeps only the highest version; trashes older copies and updates links.
Archive Older Versions Moves all non-latest versions to the archive folder (Settings!C2) instead of deleting them. Useful for audit trails.
Repair Broken Links Re-searches Drive for every row where col G shows Not Found or a stale URL, and repairs any that can be matched by exact file name.
Clear All Diagnostics Wipes col I (KAL Name Check) and removes all row background colors, resetting the sheet to an unaudited state.

📊 Reports & View

▼
Generate Health Report Opens a modal summarizing the registry: total rows, red/yellow/green counts, missing files, finalized %, and a list of all error rows.
Export Registry to PDF Downloads the Registry sheet as a formatted PDF, saved to your Drive root with the current date in the file name.
Toggle Compact View Shows/hides columns D, H, J, and L to reduce horizontal scrolling during data entry.
Toggle Done Rows Shows/hides all green (vFINAL) rows so you can focus on files still in progress.

⚡ Top-Level Actions

▼
Audit All Rows Validates every row: checks naming convention, Drive/entity/doc-type codes, searches Drive, updates cols E/F/G/I, and applies status colors. Runs as a single batch for performance.
Audit Selected Row Same as above but only for the cursor row — much faster for a quick single-row check.
Search Missing KAL Files Scans Drive for files starting with a registered Drive prefix (OP-, LP-, PC-, etc.) that are not yet in the Registry. Found files are inserted below their section in sorted order.
Notify Owner Sends an email to the address in col M for the selected row, with the row's current audit status and a link to the file. Requires Gmail send permission.

🏷️ Show Codes

▼
Show Drive Codes Lists all valid Drive prefix codes with descriptions, read live from the CODES & Descriptions tab.
Show Entity Codes Lists all valid Entity codes with descriptions (e.g. KAL = Kal Academy).
Show Doc Type Codes Lists all valid Document Type codes (e.g. POL = Policy, RPT = Report, FRM = Form).

🧠 The Metadata Packet

▼

In a traditional system, a filename is just a label. In the KAL System, a filename is code. When the AI reads a structured name it performs instant decompression of a high-density metadata packet:

20260512_OP-KAL_STRAT_TheKijiModel_v3
SegmentSignal decoded by AI
OPGeographic anchor → Operations & HR dept.
KALCorporate identity → KAL Academy voice
STRATTonal trigger → visionary, strategic
TheKijiModelToken split → "Kiji" + "Model" keywords
v3Temporal context → draft authority level

🎯 1 — Semantic Feature Extraction

▼

Structured naming provides the features (signals) that tell the AI how to think with 100% clarity:

🗺️ The "Where" — Drive Code OP acts as a geographic anchor — "You are in the Operations & HR department."
🏢 The "Who" — Entity Code KAL or PC provides corporate identity, preventing the AI from producing generic, off-brand content.
📄 The "What" — DocType STRAT → visionary language. POLICY → formal, compliance-driven. The DocType is the AI's tonal switch.

⏱️ 2 — CamelCase & Temporal Intelligence

▼

CamelCase Token Splitting

Humans use spaces; AI uses CamelCase. TheKijiModel lets the AI split tokens efficiently, recognising "Kiji" and "Model" as distinct, high-value keywords — impossible with a spaced filename.

Why it matters: Spaces break metadata parsing. CamelCase preserves human readability and machine precision simultaneously.

Date & Version Authority

A standardised date (YYYYMMDD) and version create Temporal Context:

  • vFINAL → definitive, authoritative abstract tone
  • v1 → exploratory, provisional abstract tone
  • Date → correct chronological ordering across the registry

⚡ 3 — The "AI Skip": Efficient Intelligence

▼

One of the most powerful features of the KAL System is knowing when not to work.

State Recognition
Rows marked 🟢 or vFINAL are identified as complete.
The Skip Logic
Finalised files are bypassed during bulk syncs — their abstracts are not regenerated.
The payoff: Custom-edited abstracts are preserved · API costs are slashed · Hundreds of files audited in seconds, not minutes.

🛡️ 4 — Hallucination Control

▼

AI hallucinations occur when a model has too much creative freedom. The KAL System eliminates this by design.

❌ Without KAL "This document appears to be related to some kind of organisational strategy…"
✅ With KAL "This KAL Operations STRAT document presents the Kiji Model framework…"

By providing verified codes from your Settings, you build a Grounded Boundary. The AI's only job is to expand your verified organisational logic into natural language — not invent it.
