📋 What is the KAL Registry?

▼

The KAL Smart File Registry is a Google Sheets-based document control system that tracks every KAL Academy file across Google Drive.

It enforces a strict naming convention, validates each file against Drive, assigns visual audit status, and automates file creation, archiving, linking, and reporting — all from a single spreadsheet menu.

Sheets used: Registry (main data) · CODES & Descriptions (validation) · Settings (config) · Temp (scratch)

🚀 Quick Start

▼
  1. Open the Registry tab — each row is one file.
  2. Fill in col B (Description) and col C (KAL File Name).
  3. Choose col D (File Type) from the drop-down.
  4. Set col E (Version) e.g. v1, v2, vFINAL.
  5. Run KAL Registry › Audit All Rows to validate the sheet.
  6. Fix any red-highlighted rows shown by the audit.

🎨 Row Status Colors

▼

After running an audit, each row is colored to show its status:

Red — Naming Error (Priority 1)
The file name fails structural or code validation. The Kiji Check column (I) shows the exact error(s), e.g. Invalid Drive Code (XX).
Yellow — File Not Found (Priority 2)
Name is valid but no matching file was found in Google Drive. Create the file or repair the link.
Green — Finalized (Priority 3)
Name is valid, file exists in Drive, and version is vFINAL.
No color — In Progress
Name is valid and file exists, but not yet finalized.

⚙️ Settings Tab Requirements

▼

The Settings sheet must have these cells configured before using advanced features:

A2 Default Destination Drive folder URL (used by Move to Destination Drive)
B2 Preferred KAL Template folder URL (used by Create Selected File)
C2 Archive folder URL (used by Archive Older Versions and Promote to Final)
Required: Paste the full Google Drive folder URL (not just the ID) into each cell. Functions will throw an error if the cell is empty.

📐 KAL Naming Convention

▼
[DRIVE]-[ENTITY]_[DOCTYPE]_[Description]_[YYYYMMDD]_v[N]
Segment Rules Example
DRIVE 2–3 char code from CODES tab OP, LP, PC
ENTITY Org unit code from CODES tab KAL, STU
DOCTYPE Document type from CODES tab POL, RPT, FRM
Description CamelCase, no spaces OnboardingGuide
Date YYYYMMDD (optional) 20240315
Version v1, v2 … vFINAL v3, vFINAL

Full example:

OP-KAL_POL_DataPrivacyPolicy_20240315_vFINAL
Separator rules: Use - (hyphen) after the Drive code, _ (underscore) between all other segments. No spaces anywhere.

📊 Registry Column Reference

▼

The Registry sheet has 13 columns (A–M). Columns shown in AUTO are script-managed; MANUAL are user-entered.

Col Field Notes
A AUTO
Row #		 Sequential row number; updated by audit.
B MANUAL
Description		 Human-readable name for the file. Used in Abstract formula.
C MANUAL
KAL File Name		 Full KAL-format name without extension. Validated on audit.
D MANUAL
File Type		 Drop-down: Google Doc, Google Sheet, PDF, etc.
E AUTO
Current Version		 Extracted from file name (e.g. v2, vFINAL).
F AUTO
Current Folder		 Drive folder name where file currently lives.
G AUTO
Link		 Clickable hyperlink to the file in Google Drive. Shows Not Found if missing.
H MANUAL
For Who		 Audience tag (staff, student, public, etc.).
I AUTO
KAL Name Check		 Audit result. Blank = pass. Error text = fail (row goes red).
J MANUAL
Destination Drive		 Target folder URL for Move to Destination. Overrides Settings!A2.
K MANUAL
Preferred Template		 Template ID or name used by Create Selected File.
L AUTO
Abstract		 AI-generated two-sentence summary using the =AI() formula.
M MANUAL
Owner Email		 Email address for Notify Owner function. Optional.

🤖 Abstract Column (L)

▼

Column L is auto-populated with an =AI() formula whenever a valid file name exists in col C:

=AI("Based ONLY on description '"&B2&"' and filename '"&C2&"', write a two-sentence summary.")

The formula uses the Gemini AI extension built into Google Sheets. If the cell shows an error, ensure the AI extension is installed and enabled for your account.

The Abstract is the rightmost data column intentionally — it can be hidden during data entry without losing any registry data.

📁 File Operations

▼
Create Selected File Creates a new Google file in Drive for the selected row using the name in col C. Optionally copies from the template specified in col K or Settings!B2. Inserts the Drive link into col G.
Open Current Folder Opens the Drive folder listed in col F for the selected row in a new browser tab.
Move to Destination Drive Moves the selected file from its current location to the URL in col J (or Settings!A2 if col J is empty). Updates col F and col G automatically.
Promote to Final & Move Renames the file to vFINAL, moves it to the Destination Drive, and archives the previous versioned copy to the archive folder (Settings!C2). Triggers a full row re-audit.
Remove Selected File Permanently deletes the Drive file linked in col G for the active row. Clears col G. Prompts for confirmation before deletion.
Remove All Versions Deletes all Drive files whose name matches the base name in col C (all version suffixes). Clears all matching links. Use with caution.

🔧 Maintenance

▼
Keep Only Latest Version Scans Drive for all versions of each registered file. Keeps only the highest version number; trashes older copies. Updates links accordingly.
Archive Older Versions Moves all non-latest versions to the archive folder defined in Settings!C2 instead of deleting them. Useful for audit trails.
Repair Broken Links Re-searches Drive for every row where col G shows Not Found or contains a stale URL, and repairs any that can be matched by exact file name.
Clear All Diagnostics Wipes col I (KAL Name Check) and removes all row background colors, resetting the sheet to a clean, unaudited state.

📊 Reports & View

▼
Generate Health Report Opens a modal dialog summarizing the registry: total rows, red/yellow/green counts, missing files, finalized %, and a list of all error rows by row number.
Export Registry to PDF Uses the Google Sheets export API to download the Registry sheet as a formatted PDF. The file is saved to your Drive root with the current date in the name.
Toggle Compact View Shows/hides columns D (Type), H (For Who), J (Destination), and L (Abstract) to reduce horizontal scrolling during data entry.
Toggle Done Rows Shows/hides all green (vFINAL) rows so you can focus on files still in progress.

⚡ Top-Level Actions

▼
Audit All Rows Validates every row in the Registry: checks naming convention, Drive code, entity, doc type, searches Drive for each file, updates col E (version), col F (folder), col G (link), col I (errors), and applies status colors. Runs as a single batch for performance.
Audit Selected Row Same as above but only for the row where your cursor is. Much faster for a quick single-row check.
Search Missing KAL Files Scans Google Drive for files whose names start with a registered Drive prefix (OP-, LP-, PC-, etc.) that are not in the Registry. Found files are inserted below their relevant section in the correct sorted position. Drive codes are read dynamically from the CODES tab.
Notify Owner Sends an email to the address in col M (Owner Email) for the selected row, with a summary of the row's current audit status and a link to the file. Requires Gmail send permission.

🏷️ Show Codes Submenu

▼
Show Drive Codes Pops a dialog listing all valid Drive prefix codes (e.g. OP, LP, PC) with their descriptions, read live from the CODES & Descriptions tab.
Show Entity Codes Pops a dialog listing all valid Entity codes with descriptions (e.g. KAL = Kal Academy, STU = Student).
Show Doc Type Codes Pops a dialog listing all valid Document Type codes (e.g. POL = Policy, RPT = Report, FRM = Form).
